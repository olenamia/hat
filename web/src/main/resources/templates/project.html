HAT (Housing Analytics Tool) provides an API for housing statistics and analytics, crucial for understanding market health and trends.

Technical Features and Details:
  • The project utilizes a multi-modular, 3-tier architecture for organized development and clear separation of responsibilities.
  • The application employs an automated ETL process to provide updated data and integrates the OpenCsv library for handling CSV data.
  • The API supports CRUD operations and performs analytics calculations.
  • Security is based on API key authentication for modification operations.
  • Swagger UI is used for user-friendly API documentation, interactive exploration, and Try It Out functionality.

Project Tech Stack:
  • Server-Side Framework: Java, Spring Boot, JPA/Hibernate

  • Database: Postgres

API Used:

The HAT Application connects to the Zillow endpoint to retrieve and process housing data. Additionally, HAT provides its own analytics API and showcases its usage through a demo, utilizing the built API.

MVP

The Minimum Viable Product (MVP) for the Housing Analytics Tool focuses on essential features, which include:

  • A dedicated tool that efficiently retrieves and processes data from the Zillow endpoint.
  • Basic analytics that present the existing state of house values and the growth of house values over different periods of time.
  • An API that responds to user requests, delivering relevant data and allowing authorized users to create, update, and delete data.
  • Security is based on API key authentication for modification operations.
  • The user interface demonstration displays current home value trends on a map and in a table view. It also shows historical trend charts and offers the ability to filter data by region type (country, state, metro, county).
Architecture Overview

The Minimum Viable Product (MVP) for the Housing Analytics Tool focuses on essential features, which include:

  1. Standalone Application: Initiates the data retrieval process from Zillow Datapoint and seeds the database.
  2. API:
    • Presents Swagger UI with API documentation.
    • Provides access to the Housing Analytics Tool (HAT) API for retrieving analytics.
    • Requires API Key authentication for data modifications.
  3. Demo User Interface (UI):
    • Built with Thymeleaf.
    • Utilizes the HAT API to retrieve analytics.
    • Demonstrates how the calculated analytics can be used.

Note:

  • Business logic and data access layers are shared across projects that provide the analytics API and database seeder.
  • The API is open for read operations, but modifications to the data require authentication via an API Key.
